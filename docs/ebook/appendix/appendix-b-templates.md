# Appendix B｜模板包

> 以下四個模板可以直接複製使用。根據你的實際需求調整 S（情境）和 I（輸入）即可。不用全部記住，用到哪個就翻哪個。

---

## 模板 1｜會議紀錄（STRIKE 版）

適合每週都在跟會議紀錄搏鬥的你。

```
S（情境）：這是一場 [部門/主題] 的會議，與會者有 [人數] 人，
          會議時長約 [X] 分鐘。

T（任務）：請根據以下逐字稿，整理成結構化的會議紀錄。

R（角色）：你是專業的會議秘書，擅長抓重點和列 action items。

I（輸入）：[貼上逐字稿或會議筆記]

K（標準）：
- 分三個區塊：決議事項 / 待辦事項（含負責人與截止日）/ 下次會議議程
- 每個要點不超過 30 字
- 用條列式

E（範例）：
決議事項：
1. Q2 預算維持原案，不追加。
2. 新專案由王小明負責，4/15 前提交計畫書。

待辦事項：
| 事項 | 負責人 | 截止日 |
|------|--------|--------|
| 提交 Q2 計畫書 | 王小明 | 4/15 |

下次會議：4/10（四）14:00
```

## 模板 2｜週報 / 月報

適合需要定期整理工作成果、但每次都從零開始寫的人。

```
S（情境）：這是 [部門/個人] 的 [週/月] 報，報告期間為 [日期範圍]。

T（任務）：根據以下工作紀錄，整理成一份結構化的 [週/月] 報。

R（角色）：你是擅長歸納與呈現成果的職場顧問。

I（輸入）：[貼上本週 / 本月的工作紀錄、筆記、成果清單]

K（標準）：
- 分四個區塊：本期成果 / 關鍵數據 / 遇到的問題與解法 / 下期計畫
- 成果用「動詞 + 具體產出 + 影響」格式
- 總長度不超過 500 字

E（範例）：
本期成果：
- 完成客戶 A 提案簡報，獲得正面回饋，預計下週簽約
- 建立會議紀錄 AI 流程，每週節省約 3 小時

關鍵數據：
- 本週完成任務：12 / 15（達成率 80%）
- 客戶滿意度回覆：4.5 / 5
```

## 模板 3｜數據分析提問

適合需要從數據中找洞察，但不知道怎麼問 AI 的人。

```
S（情境）：我有一份 [數據類型] 的資料，涵蓋 [時間範圍 / 範圍]，
          目的是 [要回答什麼問題 / 給誰看]。

T（任務）：請分析這份數據，找出關鍵發現並給出建議。

R（角色）：你是資深商業分析師，擅長從數據中提取可行動的洞察。

I（輸入）：[貼上數據或上傳檔案]

K（標準）：
- 產出 3 個關鍵發現 + 2 個可執行建議
- 每個發現要附上數據佐證
- 用非技術人員也能理解的語言
- 如果有適合的圖表，請一併產出

E（範例）：
發現一：北區 Q1 營收成長 15%（$2.3M → $2.65M），主因是新客戶數
增加 22%。建議加大北區行銷投入。

發現二：南區退貨率達 8%（全公司平均 3%），需調查品質或物流問題。
```

## 模板 4｜簡報大綱

適合每次做簡報都不知道從哪一頁開始的人。

```
S（情境）：我要做一份 [主題] 的簡報，對象是 [聽眾]，
          場合是 [會議類型]，時間約 [X] 分鐘。

T（任務）：請幫我規劃簡報大綱，包含每頁標題和要傳達的重點。

R（角色）：你是簡報顧問，擅長用清晰的邏輯結構說服聽眾。

I（輸入）：[貼上你目前的想法、素材、數據]

K（標準）：
- 總頁數控制在 [X] 頁以內
- 每頁一個核心訊息
- 用「問題→分析→方案→行動」的說服結構
- 附上每頁的講稿要點（3 句話以內）

E（範例）：
第 1 頁｜封面
第 2 頁｜問題：我們的客戶流失率在上升
第 3 頁｜數據：Q1 流失率 12%，較去年同期上升 3%
第 4 頁｜原因分析：三個主要因素
第 5 頁｜方案：建議的三個改善行動
第 6 頁｜預期效果與時程
第 7 頁｜下一步行動
```
