# Chapter 6｜任務→工具矩陣：不要追新，追「可交付」

> 這一章幫你解決「工具太多不知道選哪個」的焦慮。我們不只列工具，而是根據你的路線（A/B/C/D），幫你找到最值得投入的那幾個——然後用到熟。

---

## 為什麼不列「2026 年 Top 10 AI 工具」？

因為工具每三個月就換一輪。你今天收藏的清單，三個月後一半會改版、兩個會消失。

你可能正處在這樣的階段：手機裡存了一堆 AI 工具截圖，電腦裝了好幾個試用版，但每次真的要做事的時候，還是不確定該打開哪一個。

**真正不會過期的，是「任務分類 + 你的路線」。**

先確認你是哪條路線（Ch2），再看那條路線最常做什麼任務，最後才選工具。選的時候看四件事：社群活不活（卡住有人問）、能不能 E2E 跑通（不用跳來跳去）、成本划不划算、有什麼獨特優勢。

## 重點類別與路線建議

### AI 助手：你的主力引擎

不管哪條路線，AI 助手都是你每天開的第一個工具。差別在用法——A 拿它寫 Email 和摘要，B 拿它寫文章和企劃，C 拿它分析數據，D 拿它寫提案和交付物。

不只有美國的 ChatGPT 和 Claude。中國的 **DeepSeek** 完全免費、數學推理準確率 90%、API 成本僅 ChatGPT 的 1/10；法國的 **Mistral Le Chat** 原生支援歐洲隱私法規，Pro 方案 $15/月比 ChatGPT 便宜 25%。

對預算有限的人來說，DeepSeek + NotebookLM 的免費組合，已經能覆蓋大多數文字和研究任務。

### E2E 工作站與深度研究

**NotebookLM** 是目前最被低估的工具之一。它免費、可上傳 50 份文件做交叉分析，還能自動產出 AI 播客（兩位 AI 主持人討論你的文件內容，自然到會互相打斷）。對 **路線 B** 想做作品的人，這是免費的研究利器。

**Skywork AI** 則是多格式產出的 E2E 冠軍——文件、簡報、試算表、播客、網頁都在一個平台內完成。對 **路線 D** 的接案者來說，一站完成多格式交付物，效率差很多。

### 數據分析：路線 C 的核心戰場

如果你是路線 C，第一步不是學 Python，而是把數據丟進 Code Interpreter，用 STRIKE 告訴它你要什麼分析。對一個行銷專員來說，這樣做一份「上月活動成效分析」只需要 30 分鐘，而不是三天。**路線 A** 如果每週花時間手動做報表，Sheets + Gemini 讓你在試算表裡直接用 AI 輔助。

### 文字→視覺化：Napkin AI

多數人做完分析或寫完文章後，還要再開一個設計工具做圖。**Napkin AI** 讓你直接貼文字就出圖——流程圖、心智圖、資訊圖、商業框架圖都行，500 萬+用戶，免費方案每週 500 次足夠日常使用。四條路線都用得到。

### 自動化：n8n 為什麼領先

來自德國的 **n8n** 是目前全球社群最活躍的自動化平台：GitHub 40,000+ Stars、23 萬活躍用戶、5,000+ 社群模板。更關鍵的是它內建 70+ AI/LangChain 節點——是三大自動化工具中唯一真正為 AI Agent 設計的。可自架（資料不外流），複雜邏輯無上限。

**路線 A** 的核心任務就是自動化重複工作。先用 Make 入門（視覺化拖拉，$9/月起），等你準備好再升級 n8n。**路線 D** 則用 n8n 把服務流程標準化，每次接單都省掉重複步驟。

### AI App 建構：Lovable、Bolt、v0

這是 2025–2026 年成長最快的新類別。瑞典的 **Lovable** 兩個月就做到 $20M 年營收，全端 App 產出速度業界最快。美國的 **Bolt.new** 在瀏覽器內零設定開發，月訪問 900 萬+。**v0** 則專注在 React UI 元件品質，適合前端需求。

主要服務 **路線 D**（把 App 開發包裝成服務）和 **路線 B**（用可互動的作品展示能力）。

## 你的路線工具包

多數人以為要學很多工具才能開始，但真正拉開差距的是你有沒有用一個工具跑通一個完整的成果。以下是每條路線的優先工具，從第一個開始用就好。

### 路線 A｜職場效率派
> 先把每週省出 3 小時。

| 優先順序 | 工具 | 用來做什麼 |
|---------|------|-----------|
| 1（必備） | ChatGPT 或 Gemini | Email、摘要、紀錄、報告草稿 |
| 2（核心） | Make 或 n8n | 把重複流程自動化 |
| 3（加分） | Napkin AI | 快速把文字結論變成流程圖 |

### 路線 B｜轉職作品派
> 先做一個能被看見的作品。

| 優先順序 | 工具 | 用來做什麼 |
|---------|------|-----------|
| 1（必備） | Claude 或 ChatGPT | 研究、寫作、結構化產出 |
| 2（核心） | NotebookLM | 深度研究、交叉分析 |
| 3（核心） | Gamma | 快速做簡報作品 |
| 4（加分） | Canva | 作品集頁面設計 |

### 路線 C｜數據實戰派
> 先做一份能講清楚的分析報告。

| 優先順序 | 工具 | 用來做什麼 |
|---------|------|-----------|
| 1（必備） | ChatGPT Code Interpreter | 上傳數據、分析、出圖 |
| 2（核心） | NotebookLM | 研究背景、找數據來源 |
| 3（核心） | Napkin AI | 數據結論→資訊圖 |

### 路線 D｜接案變現派
> 先把技能包成可賣的服務。

| 優先順序 | 工具 | 用來做什麼 |
|---------|------|-----------|
| 1（必備） | ChatGPT 或 Claude | 提案、報告、交付物撰寫 |
| 2（核心） | Skywork AI 或 Gamma | 多格式交付物一站搞定 |
| 3（加分） | n8n | 服務流程自動化 |
| 4（視服務） | Kling / Lovable / 其他 | 影片用 Kling，App 用 Lovable |

## 怎麼用這張矩陣

```
1. 我是哪條路線？（A / B / C / D）
        ↓
2. 翻到「你的路線工具包」，看優先順序
        ↓
3. 從 #1（必備）開始，先用免費版跑通一個任務
        ↓
4. 確認適合 → 付費升級 / 加下一個工具
   不適合 → 換同類別的替代方案
        ↓
5. 一個任務用一個工具，用到熟再換
```

**原則：先選路線，再選工具。一個任務用一個工具，用到熟再換。**
