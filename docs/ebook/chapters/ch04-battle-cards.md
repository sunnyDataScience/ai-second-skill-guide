# Chapter 4｜四條路線作戰卡：做什麼 / 不做什麼 / 怎麼驗收

> 每條路線一頁，像作戰手冊。找到你的路線，撕下來貼牆上。

---

## 路線 A｜職場效率派

**最小成果：** 建立 1 個 AI 輔助的日常工作流，每週穩定省出 3 小時。

### 你只做 3 件事
1. 盤點你每週最花時間的 3 個重複任務
2. 挑其中 1 個，用 AI 建立「輸入→處理→輸出」流程
3. 連續用 2 週，修到穩定可用

### 你不要做 3 件事
1. 不要一次改所有工作流程——先攻一個就好
2. 不要花時間比較工具——先用手邊有的（ChatGPT / Copilot 就夠）
3. 不要追求完美輸出——能用、能省時間就算成功

### 你會用到的工具類別
- 文字生成工具（摘要、Email、紀錄）
- 自動化串接工具（觸發 → 處理 → 通知）

### 驗收標準
- [ ] 有一個明確的工作流（能寫成 3 步驟）
- [ ] 實際使用超過 2 週
- [ ] 每週可量化省下的時間（例：3 小時）

## 路線 B｜轉職作品派

**最小成果：** 完成 1 個可放進作品集的完整作品。

### 你只做 3 件事
1. 選一個你有興趣的主題，用 AI 輔助做研究
2. 產出一個完整作品（文章 / 分析報告 / 簡報 / 企劃）
3. 把作品整理成可展示格式（PDF / 連結 / 履歷 bullet）

### 你不要做 3 件事
1. 不要想「等我學更多再做」——邊做邊學最快
2. 不要做半成品——一個完成的作品勝過十個草稿
3. 不要只放在電腦裡——發出來才算數（LinkedIn / 部落格 / 作品集網站）

### 你會用到的工具類別
- 文字生成工具（研究、撰寫、編輯）
- 簡報 / 視覺工具（排版、圖表）
- 數據工具（如果作品需要數據支持）

### 驗收標準
- [ ] 作品有明確主題、結構、結論
- [ ] 能用 3 句話說明這個作品展示了什麼能力
- [ ] 已發布或放進作品集（有連結可分享）

---

## 路線 C｜數據實戰派

**最小成果：** 完成 1 份有結論、有建議的數據分析報告。

### 你只做 3 件事
1. 找一個公開資料集（或你工作中的數據）
2. 用 AI 輔助清洗、分析、視覺化
3. 寫成一份「有結論、有建議」的報告

### 你不要做 3 件事
1. 不要從學 Python 開始——先用 AI + 試算表就能做分析
2. 不要追求複雜模型——能回答問題的簡單分析 > 看不懂的複雜分析
3. 不要只做圖表——沒有結論的圖表是裝飾品

### 你會用到的工具類別
- 數據分析工具（清洗、統計、圖表）
- 文字生成工具（解讀、撰寫報告）
- 試算表 / BI 工具（整理、呈現）

### 驗收標準
- [ ] 報告有明確的分析問題
- [ ] 有數據支持的結論（不是感覺）
- [ ] 有可執行的建議（讀的人知道下一步該做什麼）

---

## 路線 D｜接案變現派

**最小成果：** 定義 1 個最小可行服務，完成首次交付。

### 你只做 3 件事
1. 盤點你現有能力，找到 1 個「別人會需要、你能用 AI 加速」的交集
2. 定義清楚：做什麼、交什麼、多久交、多少錢
3. 找到第一個客戶（朋友、社群、免費試做都算），完成交付

### 你不要做 3 件事
1. 不要等到「準備好」才開始——第一單可以免費或超低價
2. 不要包太大——一個明確的交付物 > 模糊的「全包」
3. 不要只想不做——定義完服務就立刻找人試

### 你會用到的工具類別
- 文字生成工具（提案、報告、交付物撰寫）
- 自動化工具（流程標準化）
- 簡報 / 文件工具（交付物包裝）

### 驗收標準
- [ ] 有明確的服務定義（一句話能說清楚）
- [ ] 完成至少 1 次交付（含免費）
- [ ] 有客戶回饋（哪怕只是一句話）
